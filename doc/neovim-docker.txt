*neovim-docker.txt* Docker management for Neovim

==============================================================================
INTRODUCTION                                           *neovim-docker*

neovim-docker provides async native-buffer Docker pages for containers, images,
volumes, networks, Compose projects/services, registry workflows, and live logs.

==============================================================================
COMMANDS                                               *neovim-docker-commands*

:DockerDashboard       Open the Docker dashboard.
:DockerDashboard!      Open the Docker dashboard in a floating window.
:DockerContainers      List containers.
                       Compose containers are grouped by project when Compose
                       labels are available.
:DockerImages          List images.
:DockerVolumes         List volumes.
:DockerNetworks        List networks.
:DockerCompose         List Compose services for the current directory.
:DockerComposeCwd [dir]
                       List Compose services from a chosen working directory.
:DockerComposeProjects
                       List Compose projects discovered from Docker labels.
:DockerComposeFiles [dir]
                       Discover Compose files in a directory.
:DockerComposeContainers {project}
                       List containers for a Compose project.
:DockerRegistries      Show registry workflow commands.
:DockerRegistryStatus  Show registry config status from Docker CLI state.
:DockerLogs {name}     Tail logs for a container.
:DockerExec {name} [shell]
                       Open an interactive shell in a container.
:DockerPull {image}    Pull an image.
:DockerPush {image}    Push an image.
:DockerTag {src} {dst} Tag an image.
:DockerSearch {query}  Search Docker Hub images.
:DockerImageHistory {image}
                       Show image history/layers.
:DockerImagePrune      Prune dangling images.
:DockerAction {name} {target}
                       Run an internal action.
:DockerTelescopeContainers
                       Pick containers with Telescope if installed.
:DockerTelescopeImages
                       Pick images with Telescope if installed.
:DockerTelescopeVolumes
                       Pick volumes with Telescope if installed.

CONFIGURATION                                           *neovim-docker-config*
>
  require("neovim-docker").setup({
    docker_cmd = "docker",
    compose_cmd = { "docker", "compose" },
    log_tail = 200,
    log_max_lines = 5000,
    highlights = {
      logs = {
        error = "DiagnosticError",
        warn = "DiagnosticWarn",
        info = "DiagnosticInfo",
        debug = "Comment",
        trace = "Comment",
        success = "DiagnosticOk",
        npm_notice = "Special",
        timestamp = "Comment",
        source = "Identifier",
        http_2xx = "DiagnosticOk",
        http_3xx = "DiagnosticInfo",
        http_4xx = "DiagnosticWarn",
        http_5xx = "DiagnosticError",
      },
    },
    timeout = 30000,
    refresh_interval = 0,
    ui = {
      open_strategy = "current",
      open = nil,
    },
    keymaps = {
      enabled = true,
    },
  })
<

Docker pages are listed nofile buffers with unique meaningful docker:// names
by default, so bufferline/tabline plugins that show listed buffers can display
them. Parent app configs that hide nofile buffers or custom URI schemes may
need filters adjusted. Use ui.open to route Docker buffers through a preferred
tab or buffer plugin. Custom ui.open hooks should display the buffer
synchronously so Docker navigation can track the shown window. Return false
when the hook intentionally declines to open the buffer; that page will not be
added to navigation history. log_tail controls the initial docker logs --tail
value. log_max_lines bounds the retained live log output lines in memory while
docker logs --follow runs. Invalid values fall back to the default cap.
Live log buffers consume ANSI color codes from container output and render
those colors without showing raw escape characters. Lines without ANSI colors
fall back to semantic highlighting for severity words, npm notice output,
timestamps, service prefixes, success states, and standalone 2xx-5xx
HTTP-style status numbers.
Override highlights.logs to map semantic categories to colorscheme groups.

==============================================================================
BUFFER KEYS                                             *neovim-docker-keys*

r       Refresh the page.
/       Filter rows.
x       Clear the active filter.
o       Cycle sort columns.
a       Open the action menu.
?       Open page help.
CTRL-O  Move back through Docker pages/details/help/logs when still open.
CTRL-I  Move forward through Docker pages/details/help/logs when still open.
b       Go back from the help page.
i       Inspect/open details.
ENTER   Inspect/open details, or expand/collapse Compose project rows in the
        container page.
l       Tail logs where supported.
e       Exec into a container where supported.
s/S/R   Start, stop, or restart where supported.
d       Remove/down where supported.
p       Prune where supported.
q       Close the page.
